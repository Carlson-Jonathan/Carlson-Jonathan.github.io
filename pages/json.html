Now that your student objects are created, we will introduce another function that inherits the objects we just used and adds two additional properties- age and gender. Though it is a bit difficult to tell from this example, the new object does in fact contain all the properties of the previous object because of the inheritence. You can tell this by looking at the source code and testing by uncommenting the document.write() function, or by simply clicking on the lower buttons before the initial object is created to see that they don't work until they have something to inerit from.

We will now stringify our objects using JSON. Interestingly, when I first tried to stringify the above objects, the results only showed the object without its parent object properties. I researched the problem and found an answer on StackOverflow.com. Aparently "the author [of JSON] has not only documented the problems, but also written a library called HydrateJS that helps [to correct this issue]." To solve this problem, I created another function constructor ("flatten") to accept our cStudent objects as parameters, then simply accepts each property as part of itself.
